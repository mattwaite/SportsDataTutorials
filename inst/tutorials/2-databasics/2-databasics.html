Rows and columns

Data, oversimplifying it a bit, is information organized. Generally speaking, it‚Äôs organized into rows and columns. Rows, generally, are individual elements. A team. A player. A game. Columns, generally, are components of the data, sometimes called variables. So if each row is a player, the first column might be their name. The second is their position. The third is their batting average. And so on.

One of the critical components of data analysis, especially for beginners, is having a mental picture of your data. What does each row mean? What does each column in each row signify? How many rows do you have? How many columns?

Types

There are scores of data types in the world, and R has them. In this class, we‚Äôre primarily going to be dealing with data frames, and each element of our data frames will have a data type.

Typically, they‚Äôll be one of four types of data:

  • Numeric: a number, like the number of touchdown passes in a season or a batting average.
  • Character: Text, like a name, a team, a conference.
  • Date: Fully formed dates ‚Äì 2019-01-01 ‚Äì have a special date type. Elements of a date, like a year (ex. 2019) are not technically dates, so they‚Äôll appear as numeric data types.
  • Logical: Rare, but every now and then we‚Äôll have a data type that‚Äôs Yes or No, True or False, etc.

Question: Is a zip code a number? Is a jersey number a number? Trick question, because the answer is no. Numbers are things we do math on. If the thing you want is not something you‚Äôre going to do math on ‚Äì can you add two phone numbers together? ‚Äì then make it a character type. If you don‚Äôt, most every software system on the planet will drop leading zeros. For example, every zip code in Boston starts with 0. If you record that as a number, your zip code will become a four digit number, which isn‚Äôt a zip code anymore.

A simple way to get data

One good thing about sports is that there‚Äôs lots of interest in it. And that means there‚Äôs outlets that put sports data on the internet. Now I‚Äôm going to show you a trick to getting it easily.

The site sports-reference.com takes NCAA (and other league) stats and puts them online. For instance, here‚Äôs their page on Nebraska basketball‚Äôs game logs, which you should open now.

Now, in a new tab, log into Google Docs/Drive and open a new spreadsheet. In the first cell of the first row, copy and paste this formula in:

If it worked right, you‚Äôve got the data from that page in a spreadsheet.

Now you have data, but your headers are all wrong. You want your headers to be one line ‚Äì not two, like they have. And the header names repeat ‚Äì first for our team, then for theirs. So you have to change each header name to be UsORB or TeamORB and OpponentORB instead of just ORB.

After you‚Äôve done that, note we have repeating headers. There‚Äôs two ways to deal with that ‚Äì you could just hightlight it and go up to Edit > Delete Rows XX-XX depending on what rows you highlighted. That‚Äôs the easy way with our data.

You can use sorting to get rid of anything that‚Äôs not data. So click on Data > Sort Range. You‚Äôll want to check the ‚ÄúData has header row‚Äù field. Then hit Sort.

Now all you need to do is search through the data for where your junk data ‚Äì extra headers, blanks, etc. ‚Äì got sorted and delete it. After you‚Äôve done that, you can export it for use in R.
